# 產出月度營運報告

## 目標

掃描 `orders/` 目錄下所有已完成的確認單，聚合指定月份的營運數據，產出一份 markdown 報告（含 Mermaid 圖表）。

## 參數

- `$1`：目標月份，格式 `YYYY-MM`（如 `2025-04`）

## 執行步驟

### Step 1：掃描確認單

讀取 `orders/*-確認單.md` 的所有檔案，從每張確認單的 markdown 表格中提取以下欄位：

| 訂單類型 | ID 前綴 | 金額來源 | 日期來源 | 人數來源 |
|---------|--------|---------|---------|---------|
| 訂房 | BR | 「合計」列 | 「入住日期」 | 「人數」 |
| 訂餐 | DR | 品項明細「合計」列 | 「用餐日期」 | 「人數」 |
| 場地 | VR | 費用明細「合計」列 | 「活動日期」 | 「人數」 |

**篩選**：只納入日期落在 `$1` 月份的訂單。

每筆訂單提取：
- `order_id`：從 frontmatter 或標題
- `type`：依 ID 前綴判斷（BR/DR/VR）
- `guest`：訂房人 / 訂餐人 / 訂租人
- `date`：入住日期 / 用餐日期 / 活動日期
- `guests`：人數
- `total`：合計金額（取「**合計**」列的數字，去除千分位逗號和「元」）
- `budget_result`：「預算比對」的結果（✓ 在預算內 / ✗ 超出）

### 額外提取（用於分類分析）：

**BR 專屬**：房型名稱（如 STD-D）、每晚房價、晚數
**DR 專屬**：品項列表、低消檢查結果
**VR 專屬**：場地名稱（如 VN-GRAND）、場地費、餐標金額、加購項目列表、活動類型

### Step 2：計算統計指標

**總覽指標**：
- 總營收 = Σ(所有訂單 total)
- 訂單數 = count
- 平均客單價 = 總營收 ÷ 訂單數
- 總服務人次 = Σ(所有訂單 guests)
- 預算達標率 = 「✓ 在預算內」筆數 ÷ 總筆數 × 100%

**訂房指標**（BR）：
- 客房收入 = Σ(BR total)
- 平均房價 ADR = 客房收入 ÷ Σ(BR 晚數)
- 房型分布 = 各房型出現次數

**餐飲指標**（DR）：
- 餐飲收入 = Σ(DR total)
- 平均每人消費 = 餐飲收入 ÷ Σ(DR guests)

**場地指標**（VR）：
- 場地收入 = Σ(VR total)
- 場地使用次數 = 各場地出現次數
- 活動類型分布
- 加購項目排行

### Step 3：產出報告

依照 `templates/monthly-report.md` 模板，填入數據並產出 Mermaid 圖表。

**產出路徑**：`orders/report-{$1}.md`（如 `orders/report-2025-04.md`）

## Mermaid 圖表規則

1. **營收結構 pie chart**：訂房 / 訂餐 / 場地 三類金額佔比
2. **房型分布 pie chart**：各房型訂房次數（僅當有 BR 訂單時顯示）
3. **場地使用 xychart-beta bar**：各場地使用次數（僅當有 VR 訂單時顯示）

圖表語法範例：

```
pie title 營收結構
    "訂房" : 6400
    "訂餐" : 5060
    "場地" : 235000
```

```
xychart-beta
    title "場地使用次數"
    x-axis ["大宴會廳", "玫瑰廳", "星空露臺"]
    y-axis "次數" 0 --> 5
    bar [1, 0, 0]
```

## 邊界情況

- 若指定月份無任何確認單 → 產出空報告，註明「本月無已確認訂單」
- 若某類型（BR/DR/VR）無訂單 → 該分析區塊顯示「本月無此類訂單」，對應 Mermaid 圖表省略
- 金額計算以確認單的「合計」為準，不重新從 master-data 計算

## 品質檢查

- [ ] 所有確認單都已掃描（對照 `orders/` 目錄的 `*-確認單.md` 檔案數）
- [ ] 各類小計加總 = 總營收
- [ ] 訂單明細表的筆數 = 訂單總數
- [ ] Mermaid 圖表的數據與表格一致
