# 產生桌牌圖像 Prompt

## 輸入

讀取 `orders/$ARGUMENTS-確認單.md` 中的桌次安排表，以及 `orders/$ARGUMENTS.req.md` 的需求（風格偏好）。
若有名單檔（`guest_list` 欄位），也讀取以取得完整的 title 資訊。

## 步驟

1. **提取桌次資料**：從確認單解析桌次表，取得每桌的賓客姓名與稱謂/職稱。
2. **決定卡片風格**：
   - 讀取需求檔中的桌牌風格偏好
   - 若未指定，使用 `venues.yaml` 的 `config.card_style_default`（中國水墨風）
3. **決定產出模式**：
   - 人數 <= 50：逐人產生座位卡（每人一張）
   - 人數 > 50：僅產生桌牌（每桌一張，顯示桌名）
   - 可由使用者指定覆蓋（如「只做主桌」「只做 VIP」）
4. **產出 prompt 檔**：寫入 `orders/$ARGUMENTS-card-prompt.md`

## 風格規範對照表

| 風格 | 描述 |
|------|------|
| 中國水墨風 | 典雅中國水墨風、宣紙質感背景、書法水墨點綴、姓名居中、職稱位於姓名下方且字號較小 |
| 現代簡約 | 白色背景、無襯線字體、姓名居中粗體、職稱灰色小字、邊框極細線條 |
| 歐式古典 | 米白紙質背景、金色邊框、襯線字體、花紋角飾、姓名居中、職稱斜體 |
| 花園自然風 | 淺綠水彩背景、花草邊飾、手寫風字體、姓名居中、職稱小字 |

## 產出格式

```markdown
# 桌牌圖像生成 Prompt — {order_id}

## 角色設定

你現在是一位專業的飯店宴會視覺設計 Agent。你擁有文字對話能力，以及呼叫圖像生成引擎的權限。

## 任務目標

請批次生成「{card_style}」風格的桌牌圖片。

## 產出路徑（必遵）

- 所有生成的桌牌圖片必須存於：`orders/{order_id}-cards/`
- 檔名規則：`card-{桌次或姓名}.png`

## 風格規範

{根據風格填入具體描述}

## 執行協議

1. 讀取下方名單
2. 針對每個項目呼叫圖像生成工具
3. 確認所有圖片已存入指定路徑

## 待處理名單

{從桌次表提取的 YAML 格式名單，含 name 與 title}
```

## 品質檢查

- prompt 中的名單人數/桌數與確認單桌次表一致
- 每位賓客的 name 和 title 與名單檔一致
- 產出路徑格式正確
- 風格描述完整
