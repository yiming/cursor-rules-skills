# 規劃說明

## 目標

將原本以 Cursor Rules/Skills 實作的飯店訂單管理系統，完全改用 Claude Code 的原生機制重新設計，做出「最 Claude 的版本」。

## 原版做法（Cursor）

```
.cursor/rules/hotel-orders.mdc                   ← Rule：glob 條件觸發
.cursor/skills/hotel-order-from-req/SKILL.md      ← Skill：被 Rule 引用的 SOP
AGENTS.md                                         ← 橋接檔（讓 Claude 也能用）
```

- **Rule（.mdc）**：透過 `globs: orders/**/*.md,master-data/**` 條件觸發，AI 碰到匹配檔案時自動注入結構契約。
- **Skill（SKILL.md）**：獨立的工作流程文件，由 Rule 或 AGENTS.md 引用。
- **AGENTS.md**：橋接檔，讓 Claude Code 也能讀取 Cursor 的規則與 Skill。

## 純 Claude 版做法

```
CLAUDE.md                            ← 全域規則 + Skill 派發表
orders/AGENTS.md                     ← 訂單結構契約（純規則）
master-data/AGENTS.md                ← 主資料保護規則
skills/hotel-order-from-req.md       ← Skill：獨立的 SOP 流程
```

### 三層架構

| 層次 | 檔案 | 對應 Cursor 概念 |
|------|------|----------------|
| 全域規則 | `CLAUDE.md` | `.cursor/rules/*.mdc` 的全域部分 |
| 局部規則 | `{dir}/AGENTS.md` | `.cursor/rules/*.mdc` 的條件觸發部分 |
| 工作流程 | `skills/*.md` | `.cursor/skills/*/SKILL.md` |

### 核心改變

| 面向 | Cursor 版 | 純 Claude 版 |
|------|-----------|-------------|
| 全域規則 | `.mdc` + `AGENTS.md`（橋接） | `CLAUDE.md`（原生） |
| 條件觸發 | glob 模式自動注入 | 目錄層級自動繼承（AGENTS.md） |
| 工作流程 | `.cursor/skills/*/SKILL.md` | `skills/*.md` |
| 派發機制 | Rule 的 glob 觸發 → Skill | CLAUDE.md 的對照表 → Skill |
| 特殊格式 | `.mdc`（需 YAML front matter） | 純 Markdown |

### 設計原則

1. **零橋接**：不需要額外的相容層，所有規則都用 Claude 原生格式。
2. **目錄即作用域**：利用 AGENTS.md 的目錄繼承特性，讓規則自然地限定在對應目錄。
3. **規則與流程分離**：AGENTS.md 只管「是什麼」（結構契約），Skills 只管「怎麼做」（工作流程）。
4. **CLAUDE.md 做派發**：全域規則中明確列出「什麼指令 → 對應哪個 Skill」，取代 Cursor 的 glob 觸發。
5. **永遠生效、可預期**：CLAUDE.md 永遠被讀取，不存在「漏觸發」的可能。
