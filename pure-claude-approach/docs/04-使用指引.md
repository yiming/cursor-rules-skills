# 使用指引

## 前提

本專案使用 Claude Code 操作。確保 `pure-claude-approach/` 是你的工作目錄根目錄，這樣 `CLAUDE.md` 才會被自動讀取。

## 基本操作

### 1. 處理訂單（/process-order）

直接使用指令：

```
/process-order BR250315-001
```

或用自然語言：

> 「請處理 orders/BR250315-001.req.md，產生確認單和任務」

**流程**：
1. order-clerk 子代理（sonnet）接手
2. 讀取需求 → 載入主資料 → 計算匹配
3. 按模板產出確認單 + 任務清單
4. auditor 子代理（haiku）自動驗證
5. 回報驗證結果

**產出**：
- `orders/BR250315-001-確認單.md`
- `orders/BR250315-001-任務.md`

### 2. 查詢空房（/check-availability）

```
/check-availability 2025-04-20 2025-04-22
```

Skill 載入時自動執行 shell 指令，取得：
- 所有房間清單（即時讀取 room-list.yaml）
- 已被預訂的房號（即時掃描確認單）

然後列出指定期間的可用房間。

### 3. 建立新訂單需求

在 `orders/` 目錄下新增 `.req.md` 檔案：

```markdown
---
type: 訂房
date: 2025-04-20
id: BR250420-003
guest: 張小華
---

# 訂房需求

- 入住：2025-04-20，退房：2025-04-22（共 2 晚）
- 人數：3 位（2 大人 + 1 小孩）
- 偏好：希望有加床、含早餐
- 預算：8,000 元以內
```

**ID 格式規則**：
- `BR` = 訂房、`DR` = 訂餐、`VR` = 場地
- `YYMMDD` = 日期
- `序號` = 當日第幾單（如 `-001`、`-002`）

## 安全機制

### 信任邊界（settings.json）

以下操作被系統強制禁止，即使 AI 嘗試也會被攔截：

- 修改 `orders/*.req.md`（客戶需求唯讀）
- 修改 `master-data/*`（價格來源不可竄改）

### 分工制衡

- **order-clerk**（sonnet）：能讀寫，負責產出
- **auditor**（haiku）：只能讀，負責驗證

處理和驗證由不同角色、不同模型完成，確保產出品質。

## 規則如何運作

### 目錄級規則繼承

```
CLAUDE.md            → 全域：唯讀原則、價格真實、團隊分工
master-data/CLAUDE.md → 局部：主資料保護規則
orders/CLAUDE.md     → 局部：訂單結構契約
```

操作某個目錄時，該目錄的 CLAUDE.md 自動生效，無需 glob 匹配。

### 你不需要做的事

- 不需要告訴 Claude「請遵守規則」——CLAUDE.md 永遠被讀取
- 不需要指定模型——團隊分工已定義好各角色用什麼模型
- 不需要手動驗證——auditor 會自動比對金額和房號

## 與舊版的體驗對比

### 處理一張訂單

**舊版**（規則模式）：
```
使用者：「依這張訂單產生確認單」
Claude：（讀 CLAUDE.md → 找到派發表 → 讀 skill → 照做）
結果：一份確認單 + 一份任務清單
```

**新版**（團隊模式）：
```
使用者：/process-order BR250315-001
Claude：（觸發 skill → 委派 order-clerk → 自動委派 auditor）
結果：一份確認單 + 一份任務清單 + 驗證報告
```

差異不在結果，在過程：**有分工、有委派、有制衡**。

### 查空房

**舊版**：做不到（沒有這個功能）
**新版**：`/check-availability 2025-04-20 2025-04-22` → 即時列出可用房間

## 進階操作

### 新增 Skill

1. 在 `.claude/skills/` 下建立新目錄
2. 寫入 `SKILL.md`（含 YAML frontmatter）
3. 在 CLAUDE.md 的團隊分工表中新增一行

### 新增子代理

1. 在 `.claude/agents/` 下新增 `.md` 檔案
2. 定義角色、工具、模型、限制
3. 在 CLAUDE.md 的團隊分工表中新增一行

### 修改房型或價格

需要人工直接編輯 `master-data/room-types.yaml`（AI 被 settings.json 禁止修改）。

## 常見問題

**Q: `/process-order` 和「處理這張訂單」有什麼不同？**

A: 結果相同。`/process-order` 是正式指令，會直接載入 skill 並注入參數；自然語言則需要 Claude 從 CLAUDE.md 的分工表找到對應 skill。

**Q: auditor 驗證不通過怎麼辦？**

A: auditor 會列出具體差異。你可以要求 order-clerk 根據 auditor 的回饋重新產出。

**Q: 為什麼 auditor 用 haiku 而不是 sonnet？**

A: 稽核只需要比對數字和查表，不需要複雜推理。haiku 更快、更便宜，且 maxTurns: 5 限制了執行範圍。

**Q: settings.json 的 deny 規則可以被繞過嗎？**

A: 不可以。這是系統層級的權限控制，不是 prompt 層級的建議。即使 AI 嘗試修改被 deny 的檔案，操作會被直接拒絕。
