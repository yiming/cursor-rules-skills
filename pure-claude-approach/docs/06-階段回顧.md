## 五、訂餐新增的設計亮點

### 低消邏輯

memo-07 定義的低消機制是本專案中**第一個需要逐人判斷的商業邏輯**，
比訂房的「房價 × 晚數」複雜得多：

```
訂房：total = price × nights + Σ(option × qty × days)     ← 單一公式
訂餐：total = Σ(items) + 逐人低消檢查 + 不足時建議加點     ← 多步驟判斷
```

這驗證了 Skill 不只能套模板，還能包含條件分支邏輯。

### 關聯訂單

`related_order` 欄位讓訂餐可以關聯到訂房單，
任務清單中出現「通知房務一併安排」——這是跨訂單協作的雛型。
