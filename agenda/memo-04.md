# memo-04：Cursor Rules 與 Claude Agents/Skills 操作指引

Rules / Skills / Agents 不是「給人看的文件」——是**給 AI 看的指令**。
本文用兩張測試訂單，讓你親手驗證它們的實際功能。

---

## 機制總覽

```
你說「依這張訂單產生確認單」
        │
        ▼
┌─ 自動注入（你不需要做任何事）─────────────────────┐
│  Cursor  → 讀 .cursor/rules/hotel-orders.mdc     │
│  Claude  → 讀 AGENTS.md                          │
│  → AI 自動知道：路徑、命名、YAML 欄位、禁止事項    │
└───────────────────────────────────────────────────┘
        │
        ▼
┌─ 自動觸發（AI 比對 description 決定使用）──────────┐
│  讀 .cursor/skills/hotel-order-from-req/SKILL.md  │
│  → AI 按步驟執行：讀需求→查 master data→算價→     │
│    依模板產出確認單.md 和任務.md                    │
└───────────────────────────────────────────────────┘
```

**關鍵：你只需要說一句話，Rule/AGENTS 和 Skill 在背後自動運作。**

---

## 測試一：在 Cursor 中測試（BR250420-001）

### 步驟

1. 在 Cursor 開啟本專案
2. 開啟 `orders/BR250420-001.req.md`
3. **開一個新的 AI 對話**（確保乾淨的上下文）
4. 輸入：

```
依這張訂單產生確認單與任務清單
```

5. 等 AI 產出完成

### 預期結果

AI 應在 `orders/` 下產出兩個新檔案：

**`BR250420-001-確認單.md`** 應包含：

| 驗證項目 | 預期值 | Rule 還是 Skill 確保的？ |
|----------|--------|------------------------|
| 檔名格式 | `{id}-確認單.md` | Rule（路徑約定） |
| 訂房人 | 陳美玲 | Skill（模板有 guest 欄位） |
| 房型 | 標準雙人房 STD-D | Skill（依人數匹配，容量 2 ≥ 2） |
| 房價計算 | 3200 × 2 晚 = 6400 元 | Skill（計算邏輯 + 品質檢查） |
| 加購 | 無 | Skill（需求未提及） |
| 合計 | 6400 元 | Skill（計算邏輯） |
| 預算比對 | 6500 預算，6400 合計 → 在預算內 ✓ | Skill（模板有預算比對列） |
| 建議房號 | 301~308 之一 | Rule（禁止編造房號）+ Skill（從 room-list 篩選） |

**`BR250420-001-任務.md`** 應包含：
- 具體的 checkbox 待辦清單
- 提到客戶姓名「陳美玲」
- 入住/退房日期正確

### 如何確認 Rule 有在運作？

Rule 是「隱形的」——你看不到它被載入，但可以從**反面**驗證：
- AI **沒有**覆蓋 `BR250420-001.req.md`（Rule 禁止事項）
- AI **沒有**自己編一個房價（Rule：金額必須與 master-data 一致）
- AI **沒有**建議一個不存在的房號（Rule：房號必須在 room-list.yaml）

### 如何確認 Skill 有在運作？

Skill 的效果是「可見的」——你可以從**格式**驗證：
- 確認單是**表格格式**，不是自由敘述（Skill 模板）
- 有「預算比對」這一列（Skill 模板定義的）
- 任務清單是 **checkbox 格式**（Skill 模板）

## 測試二：在 Claude Code 中測試（BR250420-002）

### 前置設定

Claude Code 不認 `.cursor/` 目錄結構，但透過共用機制，**不需要額外複製任何檔案**：

- **AGENTS.md**（已建好）：在專案根目錄，Claude Code 啟動時自動讀取
- **Skill**：AGENTS.md 末段已加入引用，指向 `.cursor/skills/hotel-order-from-req/SKILL.md`
  - Claude Code 會依 AGENTS.md 的指示去讀取同一份 SKILL.md
  - Skill 內容只維護一份，Cursor 與 Claude Code 共用

### 步驟

1. 在本專案目錄開啟 Claude Code
2. 輸入：

```
依 orders/BR250420-002.req.md 產生確認單與任務清單
```

3. 等 AI 產出完成

### 預期結果

**`BR250420-002-確認單.md`** 應包含：

| 驗證項目 | 預期值 |
|----------|--------|
| 訂房人 | 林志偉 |
| 房型 | 家庭四人房 FAM-Q（容量 4 ≥ 人數 4） |
| 房價 | 4800 × 3 晚 = 14400 元 |
| 停車 | 300/車/日 × 3 日 = 900 元 |
| 早餐 | FAM-Q 不含早餐；至少 2 小孩 × 200 × 3 = 1200 元 |
| 合計 | ≥ 16500 元 |
| 預算比對 | 16000 預算 → **超出 ≥ 500 元**，應提示或建議替代方案 |

**這張單刻意設計成超預算**——用來驗證 Skill 的預算比對功能是否真的運作。

## 兩次測試的比對重點

跑完兩張單後，比較兩邊的產出：

| 比較項目 | 期望 | 意義 |
|----------|------|------|
| 確認單格式 | 兩邊都是**表格格式**，欄位順序一致 | 證明 Skill 模板在兩個平台都生效 |
| 金額計算 | 都與 master-data 吻合，無編造 | 證明 Rule/AGENTS 的禁止事項生效 |
| 預算比對 | 001 顯示「預算內 ✓」，002 顯示「超出」 | 證明 Skill 的計算邏輯生效 |
| 任務清單 | 都是 checkbox 格式，含客戶姓名 | 證明 Skill 模板在兩邊都一致 |
| req.md | 兩邊都**未被修改** | 證明 Rule/AGENTS 的禁止事項生效 |

## 如果產出不符預期？

| 問題 | 可能原因 | 排查方式 |
|------|----------|----------|
| 格式不是表格 | Skill 沒觸發 | 對話中問 AI「你有讀到 hotel-order-from-req 這個 skill 嗎？」 |
| 金額不對 | AI 沒讀 master-data | 對話中問 AI「room-types.yaml 裡 STD-D 的房價是多少？」 |
| 用了不存在的房號 | Rule/AGENTS 沒注入 | 檢查 Rule 的 globs 是否匹配、AGENTS.md 是否在根目錄 |
| Claude Code 沒用 Skill | AGENTS.md 未引用 Skill 路徑 | 確認 `AGENTS.md` 末段有 Skills 引用段落，且路徑正確指向 `.cursor/skills/hotel-order-from-req/SKILL.md` |

---

## 總結：它們到底做了什麼？

| 機制 | 它做的事 | 你不用做的事 |
|------|----------|------------|
| Rule (.mdc) / AGENTS.md | AI 碰到訂單檔時，**自動**知道結構規矩 | 你不用每次告訴 AI「檔案放在 orders/、格式是…」 |
| Skill (SKILL.md) | AI 收到「產生確認單」時，**自動**按模板和計算邏輯執行 | 你不用每次貼模板、教 AI 怎麼算價、提醒品質檢查 |

**一句話：Rule/AGENTS 讓 AI 自動守規矩，Skill 讓 AI 自動按流程做事。你只需要說「產生確認單」。**
