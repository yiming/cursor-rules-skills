# 產生桌牌圖像 Prompt

## 使用方式

```
@hotel-generate-card-prompt 請為 @orders/VR250420-001-確認單.md 產生桌牌 prompt
```

## 步驟

### Step 1：讀取資料

1. 讀取 `orders/{id}-確認單.md`，從「桌次安排」區塊提取每桌的賓客姓名與稱謂
2. 讀取 `orders/{id}.req.md`，取得風格偏好（若有指定）
3. 若有名單檔（`guest_list` 欄位），也讀取以取得完整 title 資訊

### Step 2：決定卡片風格

- 讀取需求檔中的桌牌風格偏好
- 若未指定，使用預設風格：**中國水墨風**

| 風格 | 描述 |
|------|------|
| 中國水墨風 | 典雅中國水墨風、宣紙質感背景、書法水墨點綴、姓名居中、職稱位於姓名下方且字號較小 |
| 現代簡約 | 白色背景、無襯線字體、姓名居中粗體、職稱灰色小字、邊框極細線條 |
| 歐式古典 | 米白紙質背景、金色邊框、襯線字體、花紋角飾、姓名居中、職稱斜體 |
| 花園自然風 | 淺綠水彩背景、花草邊飾、手寫風字體、姓名居中、職稱小字 |

### Step 3：決定產出模式

| 人數 | 模式 | 說明 |
|------|------|------|
| ≤ 50 人 | 座位卡 | 每人一張，顯示姓名 + 職稱 |
| > 50 人 | 桌牌 | 每桌一張，顯示桌名 |

使用者可指定覆蓋（如「只做主桌 VIP 座位卡」）。

### Step 4：產出 prompt 檔

寫入 `orders/{id}-card-prompt.md`，格式如下：

---

```markdown
# 桌牌圖像生成 Prompt — {order_id}

## 角色設定

你現在是一位專業的飯店宴會視覺設計 Agent。你擁有文字對話能力，以及呼叫圖像生成引擎的權限。

## 任務目標

請批次生成「{card_style}」風格的{桌牌/座位卡}圖片。

## 產出路徑（必遵）

- 所有圖片必須存於：`orders/{order_id}-cards/`
- 檔名規則：`card-{桌名或姓名}.png`

## 風格規範

{根據風格填入具體描述}

## 執行協議

1. 讀取下方名單
2. 針對每個項目呼叫圖像生成工具
3. 確認所有圖片已存入指定路徑

## 待處理名單

{從桌次表提取的 YAML 格式名單，含 name 與 title}
```

---

## Step 5：Cursor 內完成 AI-to-AI 接力

prompt 檔產出後，**直接在 Cursor 中執行**（不需要切換工具）：

```
請依照 @orders/{id}-card-prompt.md 的指示，批次生成桌牌圖片
```

建議使用 Gemini 模型（支援圖像生成）。

### 進階用法

只產主桌 VIP 個人座位卡：
```
請依照 @orders/{id}-card-prompt.md 的名單，
加上 mode: 座位卡，幫我生成主桌 {n} 位 VIP 的個人座位卡圖片
```

## 品質自查

- [ ] prompt 中的名單人數/桌數與確認單桌次表一致
- [ ] 每位賓客的 name 和 title 與名單檔一致
- [ ] 產出路徑格式正確（`orders/{id}-cards/`）
- [ ] 風格描述完整
